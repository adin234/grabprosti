GRABPROSTI*: ONE-STOP APP TO MEETING UP
Aldrin Bautista, Alexandre Gerona, Owen Sen Parcon, and Xrijslove Meneses
Abstract

This study aims to create a mobile application that will integrate lodging and transportation booking to the usual app dating experience. This integration will save logs to keep the users secured and accounted for. These logs can be checked in case of emergency and will help in pointing out where a user went and the person he/she is with during that time.

Introduction

With Internet and Smartphones taking over the way we live today, a lot of things changed. With the emergence of the smartphone, App Dating was introduced in 2003 (Gates, Hoyt, Wakefield, Stewart, & Turner, 2015) and together with Online dating introduced us to new ways of connecting and hanging out with other people. App dating takes advantage of the mobile phone's capabilities such as GPS and suggests a match using these data (ie. user's current location). The most known app of this kind is Tinder which was launched in the fall of 2012. Although, there were no police reports that links Tinder directly to crime, there are a lot of Tinder dates that often go wrong which lowers the approval rating of app dating to other prospective users. In this study, a mobile dating application will be created that has a log of transactions on matching, transportation, and accommodation for the users to have a safe correspondence.

Methods
  • A. Development Environment
    • 1. Mininum Hardware Requirements
      • A. Server
        • 8-cores
        • 4Gb RAM
        • Ubuntu 14.04
      • B. Mobile Phone
        • Android 4.0.4/iOS 7
        • 1Gb RAM (or apple equivalent)
        • 1.2Ghz dual-core processor (or apple equivalent)
    • 2. Software Requirements
      • Android SDK
      • Java Development Kit 1.7
      • Android Studio
      • Node.js
      • Express.js
      • Redis
      • MariaDB
  • B. Architecture
    • 1. System flow
    • 2. Match Creator/Finder: Processes regarding matching will take place in this module. Matches will be based mainly on location and interests of the user.
    • 3. Transportation Booking: An independent module for booking of cars for pick-up (user's location) and drop-off (meeting place).
    • 4. Lodging Booking: Like the Transportation Booking Module, this module was independently written and serves as a service for reservation of a place to "meetup" or "hangout".
    • 5. Logging: Refers to the complete log of the user’s activities.
    • 6. Feedback: This dialog box which appears on the user’s profile is a module for rating the “match” and feedbacks about their “date”.
Results and Discussion
The Mobile Application is the only way a user can create a booking, and no Web Application was provided.

Login: Asks for username and password

Signup: Asks for username, password and password confirmation

Match Finder: Finds a “match” based on the user’s location. Swiping left will move on to the next “match” and swiping right will create a prompt to the other user he/she is trying to “match” with.

Messages: This view the lists of all conversations with other users with two available action —- Send Message and Request for Meetup.

Conclusion

By using a Node.js server with Express as a framework, the server was able to maintain a quick response time to users therefore giving a fluid user experience. The user of redis as a backend cache further improved the systems' performance and stability by lessening the database transactions.

The mobile application enabled the users to feel more secured in dating and meeting strangers online. Moreover, dating was revolutionized since it is more convenient and safe for the users.

Future Work

The app still has a long way to go to fully revolutionize Online and App Dating. Users will have more options if the transportation booking module is integrated to already known services like GrabTaxi, EasyTaxi, and Uber. Aside from the transportation booking module, it will also help a lot if the lodging booking module will be integrated with AirBnb, Booking.com and other services having the same function. For convenience, it is also helpful if the app will integrate online payment gateway so as to have cashless transactions.
